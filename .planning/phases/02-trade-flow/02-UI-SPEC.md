## Color

All colors are pre-existing project tokens from `globals.css`. No new colors are introduced.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-void` | #0a0a0e | Page background |
| Secondary (30%) | `--color-abyss` / `--color-surface` | #121216 / #1a1a1e | Cards, panels, OrderBook bg, modal bg |
| Accent (10%) | `--color-star` | #e8a825 | See reserved-for list below |
| Destructive | `--color-nova` | #ef4444 | Cancel order confirmation button only |
| Positive | `--color-aurora` | #10b981 | Discounted fee display, positive rate indicators |
| Info | `--color-nebula` | #3b82f6 | Blended rate highlight, match count badge |

Accent (`star`) reserved for:
- Primary CTA buttons (Quick Lend, Sign & Submit)
- Active tab indicator on trade mode selector (Lend / Swap / Advanced)
- Focus ring on form inputs
- Hover state on PairCard pair name
- Rate highlight on best available APR in OrderBook

Never use accent for: body text, borders at rest, skeleton loaders, section backgrounds.

## Visual Hierarchy

Trade page focal-point order:

1. **Primary: Order Form** -- The main interaction area (mode tabs, token selectors, amount input, Sign button). Visually dominant via card elevation (`bg-surface`) and largest content block.
2. **Secondary: Order Book** -- Contextual pricing data. Placed below the form. Uses `bg-abyss` to recede behind the form.
3. **Tertiary: BestTradesPanel** -- Match preview and blended rate summary. Placed below the Order Book. Visually lightest; content appears only when matches exist.

## Interaction Contracts

### NAV-01: State-Preserving Navigation

| Property | Contract |
|----------|----------|
| URL params | `debtToken`, `collateralToken`, `mode`, `amount` via nuqs |
| Mode values | `lend` (default), `swap`, `advanced` |
| Amount format | Human-readable decimal string (e.g. `100.5`), not raw bigint |
| Default mode | `lend` when `mode` param absent |
| Token display | Resolve to symbol via `findTokenByAddress`; fall back to truncated address |
| Behavior on navigate | Trade page reads nuqs params on mount; form fields pre-populated; no flicker (Suspense boundary) |

### NAV-02: Shareable Trade Links

| Property | Contract |
|----------|----------|
| URL format | `/trade?debtToken={addr}&collateralToken={addr}&mode={mode}&amount={amount}` |
| Two-way binding | Changing any form field updates URL in real-time (shallow push via nuqs) |
| Partial URLs | All params optional; page renders with whatever is provided |

### NAV-03: Quick Lend from Browse

| Property | Contract |
|----------|----------|
| Trigger | "Quick Lend" button on PairCard (alongside existing "Trade" button) |
| Pattern | Modal (shadcn Dialog component) |
| Modal width | `max-w-md` (448px) |
| Modal contents | Token pair header, amount input, duration selector, FeeBreakdown, Sign button |
| Wallet gate | Wrap Sign button with `Web3ActionWrapper` (existing pattern) |
| Close behavior | Click overlay, press Escape, or click X button |
| Success behavior | Close modal, show Sonner toast: "Lend offer signed successfully" |
| Loading state | Sign button shows Loader2 spinner + "Signing..." text, disabled |
| Quick Lend button min height | 44px (accessibility touch target) |
| Modal CTA min height | 36px (consistent with existing modal button sizing) |

### TRADE-01: Order Book on Trade Page

| Property | Contract |
|----------|----------|
| Placement | Below trade form, above BestTradesPanel |
| Component | Existing `OrderBook` component (no rebuild) |
| Sort order | APR ascending (lowest/best rate first) -- already implemented server-side |
| Loading state | `BookSkeleton` (existing component) |
| Guard | Only render when both `debtToken` and `collateralToken` are set |
| Duration filter | Show duration filter tabs in lending mode (existing DurationFilter component) |

### TRADE-02: Fee Visibility

| Property | Contract |
|----------|----------|
| Trade page | FeeBreakdown already present (line 998) -- verify visible on all 3 form variants |
| Quick Lend modal | FeeBreakdown must appear above the Sign button |
| SettlementDrawer | FeeBreakdown must appear inside drawer before confirm |
| Placement rule | FeeBreakdown always appears directly above the final action button, never hidden behind a toggle |

### TRADE-03: Blended Rate Preview

| Property | Contract |
|----------|----------|
| Display location | Inside BestTradesPanel, as a summary row above the individual match rows |
| Computation | Weighted-average APR from `selectOrders` output (debt-amount-weighted) |
| Format | "{N.NN}% blended APR" in `font-mono text-sm` |
| Color | `text-nebula` (#3b82f6) to distinguish from individual row APRs |
| Empty state | Hidden entirely when no matches or single match (blended = individual, no value add) |
| Threshold | Only show when 2+ orders are selected for multi-fill |
