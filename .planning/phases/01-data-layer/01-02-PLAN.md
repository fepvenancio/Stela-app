Migrate each hook from `useFetchApi` to `useQuery`. Keep `buildApiUrl` from `api.ts` (still useful). Keep all existing type exports. Preserve the same return shape for each hook so consumers don't break.

**Pattern for each hook:**
- Replace `import { useFetchApi } from './api'` with `import { useQuery } from '@tanstack/react-query'`
- Add `import { queryKeys } from '@/lib/query-keys'`
- Keep `import { buildApiUrl } from './api'` where URL building is needed
- Replace `useFetchApi<T>(url, opts, interval)` with `useQuery({ queryKey, queryFn, refetchInterval })`
- The `queryFn` does: `const res = await fetch(url); if (!res.ok) throw new Error(\`HTTP \${res.status}\`); return res.json()`
- Use `enabled: Boolean(url)` when the URL can be null/undefined

**Hook-by-hook specifics:**

1. **usePairs.ts** — queryKey: `queryKeys.pairs.all`, refetchInterval: `15_000`. queryFn fetches `/api/pairs`, returns `json.data`. Return `{ data: data ?? [], isLoading, error, refetch }`.

2. **useInscriptionDetail.ts** — queryKey: `queryKeys.inscriptions.detail(inscriptionId)`, no refetchInterval, `enabled: Boolean(inscriptionId)`. queryFn fetches `/api/inscriptions/${inscriptionId}`, returns the FULL response object (detail + assets together). Keep the existing `InscriptionDetail` interface and the mapping from raw to detail shape. Return `{ data: detail, assets, isLoading, error, refetch }`.

3. **useInscriptionAssets.ts** — Use the SAME queryKey as useInscriptionDetail: `queryKeys.inscriptions.detail(inscriptionId)`. Use `select` to extract just the assets from the shared cache entry. Do NOT provide a separate `queryFn` — instead, use the `select` option to transform cached data client-side:
   ```typescript
   useQuery({
     queryKey: queryKeys.inscriptions.detail(inscriptionId),
     queryFn: async () => {
       const res = await fetch(`/api/inscriptions/${inscriptionId}`)
       if (!res.ok) throw new Error(`HTTP ${res.status}`)
       return res.json()
     },
     enabled: Boolean(inscriptionId),
     select: (data) => data.assets ?? [],
   })
   ```
   This shares the same cache entry as useInscriptionDetail — only one fetch happens, and `select` transforms the cached data for this hook's consumers. Return `{ data: data ?? [], isLoading, error, refetch }`.

4. **useOrderBook.ts** — queryKey: `queryKeys.orders.book(debtToken, collateralToken, options?.duration)`, refetchInterval: `options?.refreshInterval ?? 30_000`, `enabled: Boolean(debtToken && collateralToken)`. queryFn fetches the built URL. Return `{ data: data ?? null, isLoading, error: error?.message ?? null, refetch }`.

5. **useInscriptions.ts** — queryKey: `queryKeys.inscriptions.list({ status, address, page })`, refetchInterval: `15_000`. queryFn builds URL with buildApiUrl and fetches. Return type: `data ?? []`.
   - **Optimistic updates:** Replace the `stela:optimistic-create` event listener with a new exported `addOptimisticInscription` function that takes `(queryClient: QueryClient, inscription: InscriptionRow)` and calls `queryClient.setQueryData(queryKeys.inscriptions.list({ status: undefined, address: undefined, page: undefined }), (old) => [inscription, ...(old ?? [])])`.
   - The function signature is: `export function addOptimisticInscription(queryClient: QueryClient, inscription: InscriptionRow)` — callers must pass their queryClient instance.
   - Remove the `stela:optimistic-create` event listener, the `optimisticInscriptions` state, and the cleanup effect.

6. **useOrders.ts** — Two hooks:
   - `useOrders`: queryKey: `queryKeys.orders.list({ status, address, page, limit })`, refetchInterval: `30_000`. queryFn fetches with buildApiUrl. Return `{ data: data ?? [], isLoading, error, refetch }`.
   - `useOrder`: queryKey: `queryKeys.orders.detail(id)`, refetchInterval: `5_000`, `enabled: Boolean(id)`. Return `{ data, isLoading, error, refetch }`.

**For contract read hooks that LISTEN to `stela:sync`** (2 hooks):

These hooks use `@starknet-react/core`'s `useReadContract` which has its own cache separate from TanStack Query. Replace the `stela:sync` listener with a TanStack Query `onSettled` subscription approach:

1. **useInscription.ts**: Remove the `useEffect` that listens for `stela:sync` and calls `result.refetch()`. Instead, use a `useEffect` that watches for TanStack Query cache invalidations.
