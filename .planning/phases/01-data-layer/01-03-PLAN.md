**Migrate usePortfolio.ts:**

Replace the entire manual fetch/pagination/sync system with TanStack Query hooks. The categorization logic (lending/borrowing/repaid/redeemable) stays exactly the same.

Structure:
- Import `useQuery` from `@tanstack/react-query`, `queryKeys` from `@/lib/query-keys`, `buildApiUrl` from `./api`
- Keep ALL existing type exports (`EnrichedInscription`, `PortfolioData`, `OrderRow` re-export)
- Keep the same constants: `ACTIVE_STATUSES`, `INSCRIPTIONS_LIMIT = 200`, `ORDERS_LIMIT = 200`, `T1_LIMIT = 100`

Replace the 5 manual fetch patterns with 6 useQuery calls:

1. **Inscriptions query**: `useQuery({ queryKey: queryKeys.portfolio.inscriptions(address!), queryFn: async () => { fetch all inscriptions for address with limit=INSCRIPTIONS_LIMIT }, enabled: Boolean(address), refetchInterval: 30_000 })`
   - queryFn: `const url = buildApiUrl('/api/inscriptions', { address, limit: INSCRIPTIONS_LIMIT }); const res = await fetch(url); if (!res.ok) throw ...; return (await res.json() as ApiListResponse<InscriptionRow>).data`

2. **Orders query**: `useQuery({ queryKey: queryKeys.portfolio.orders(address!), queryFn: ..., enabled: Boolean(address), refetchInterval: 30_000 })`
   - queryFn fetches `/api/orders` with `{ address, status: 'all', limit: ORDERS_LIMIT }`

3. **Shares query**: `useQuery({ queryKey: queryKeys.portfolio.shares(address!), queryFn: ..., enabled: Boolean(address) })`
   - queryFn fetches `/api/shares/${address}`

4. **Collection offers query**: `useQuery({ queryKey: queryKeys.portfolio.collectionOffers(address!), queryFn: ..., enabled: Boolean(address), refetchInterval: 30_000 })`
   - queryFn fetches `/api/collection-offers` with `{ address, limit: T1_LIMIT }`

5. **Refinances query**: `useQuery({ queryKey: queryKeys.portfolio.refinances(address!), queryFn: ..., enabled: Boolean(address), refetchInterval: 30_000 })`

6. **Renegotiations query**: `useQuery({ queryKey: queryKeys.portfolio.renegotiations(address!), queryFn: ..., enabled: Boolean(address), refetchInterval: 30_000 })`

Simplification: Remove the manual pagination (loadMoreInscriptions, loadMoreOrders, hasMore) from this hook. Currently the limits are 200 which loads everything in one page for most users. The `hasMoreInscriptions`, `hasMoreOrders`, `loadMoreInscriptions`, `loadMoreOrders`, `isLoadingMoreInscriptions`, `isLoadingMoreOrders` fields should remain in the return type but return static values: `hasMore*: false`, `loadMore*: () => {}`, `isLoadingMore*: false`. This keeps the interface stable for consumers.

The categorization `useMemo` block (lines 239-364 in current file) stays EXACTLY as-is, but reads from `inscriptionsQuery.data`, `ordersQuery.data`, `sharesQuery.data`, etc. instead of the old state variables.

Remove: all `useState`, `useCallback`, `useRef`, `useEffect` blocks for fetching/pagination/sync. Remove `stela:sync` listener. Remove `setInterval` polling. Remove the `fetchInscriptionsPage` and `fetchOrdersPage` callbacks.
