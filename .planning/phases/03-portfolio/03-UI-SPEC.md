## Interaction States

### SummaryBar

| State | Behavior |
|-------|----------|
| Loading | Show 4 skeleton rectangles matching MetricCard dimensions (rounded-2xl, h-[72px]) |
| No positions | Hide SummaryBar entirely (do not show zeros) |
| Has positions | Show SummaryBar with computed metrics |

### Position Row Actions

| Status | Role | Button | Click Behavior |
|--------|------|--------|---------------|
| filled / grace_period | Borrower | "Repay" | Execute repay immediately, show Loader2 spinner while pending |
| repaid / liquidated | Lender (shares > 0) | "Claim" | Execute redeem immediately, show Loader2 spinner while pending |
| open | Creator | "Cancel" | Show inline Yes/No confirmation, then execute |
| overdue | Anyone | "Liquidate" | Show inline Yes/No confirmation, then execute |
| pending (order) | Borrower | "Cancel" | Show inline Yes/No confirmation, then call cancel API |

### Action Feedback

| Event | Feedback |
|-------|----------|
| Action submitted | Button shows Loader2 spinner, disabled state |
| Action succeeded | Toast via sonner ("Repaid successfully" / "Shares claimed" / "Position cancelled" / "Liquidation submitted") |
| Action failed | Toast via sonner with error message |
| Data refresh | TanStack Query invalidation triggers automatic refetch of portfolio data |

## Layout Contract

```
Portfolio Page Layout:
+--------------------------------------------------+
| Page Header: "Portfolio" (text-xl font-display)   |
| Search Input (existing)                           |
+--------------------------------------------------+
| SummaryBar (4-column grid, 2-col on mobile)       |
|   [Total Lent] [Total Borrowed] [Redeemable] [Orders] |
+--------------------------------------------------+
| Tabs: [Active] [Pending] [History]                |
+--------------------------------------------------+
| Tab Content:                                      |
|   Section heading (e.g. "Lending", "Borrowing")   |
|   ListingTableHeader                              |
|   PortfolioInscriptionRow (repeated)              |
|   ...                                             |
|   Section heading (e.g. "Redeemable")             |
|   PortfolioInscriptionRow (repeated)              |
|   ...                                             |
|   OR EmptyTab (if no items)                       |
+--------------------------------------------------+
```

SummaryBar placement: Between search input and tabs. Always visible when user has any positions. Hidden (not rendered) when user has zero total positions across all categories.

Source: RESEARCH.md Pattern 3 (SummaryBar Placement), existing portfolio page structure.
